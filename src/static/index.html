API Documentation

This document provides information on how to use the available endpoints in this API, including query parameters for pagination, sorting, and filtering.

1. Get Depths History

Endpoint: GET /get-depths-history

Description: Retrieves depths history with optional filtering, pagination, and sorting options.

Query Parameters

  • date_range: Date range filter in a specific format.
  • from: Start date (timestamp).
  • to: End date (timestamp).
  • liquidity_gt: Minimum liquidity filter.
  • sort_by: Field to sort by (default: timestamp).
  • order: Sorting order (asc or desc).
  • page: Page number for pagination (default: 1).
  • limit: Number of results per page (default: 10).
  • count: Total number of records to fetch (default: 30).
  • interval: Time interval between records.

Example Request

GET /get-depths-history?from=1620000000&to=1630000000&liquidity_gt=5000&sort_by=timestamp&order=desc&page=2&limit=15

2. Get Earnings History

Endpoint: GET /get-earnings-history

Description: Retrieves earnings history with filtering, sorting, and pagination options.

Query Parameters

  • Same parameters as Get Depths History (see above).

Example Request

GET /get-earnings-history?date_range=2021-01-01_to_2021-12-31&order=asc&limit=20

3. Get Swaps History

Endpoint: GET /get-swaps-history

Description: Retrieves swaps history with options for pagination, sorting, and filtering.

Query Parameters

  • Same parameters as Get Depths History (see above).

Example Request

GET /get-swaps-history?from=1620000000&sort_by=timestamp&order=desc&liquidity_gt=1000&page=1&limit=10

4. Get Rune Pool History

Endpoint: GET /get-runepool-history

Description: Retrieves history of the Rune pool with various filtering, pagination, and sorting options.

Query Parameters

  • Same parameters as Get Depths History (see above).

Example Request

GET /get-runepool-history?to=1630000000&sort_by=timestamp&order=asc&page=3&limit=5&count=50
